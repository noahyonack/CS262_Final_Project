parallelogram.parallelogram module¶

This file contains our library’s implementations of p_map(), p_filter(), and p_reduce().

We use the letter “p” because it indicates that the method is paralellized and because doing so ensures that our functions are properly namespaced

Imports helpers.py, which contains ‘private’ helper functions

parallelogram.parallelogram.p_filter(foo, data, port, timeout)[source]¶

Filter a function foo() over chunks of data (of type list) and join the filtered chunks before returning back to the caller.

This filtering will likely not be done on a single machine (unless the data to be filtered over is so small that sending it over a network would be inefficient.)

param foo:function to filter over data
Parameters:
  • data – a list of data to be filtered
  • port – a port by which to send over distributed operations
Returns:

the filtered results

parallelogram.parallelogram.p_reduce(foo, data, port, timeout)[source]¶
Reduce a function foo() over chunks of data (of type list) and

then reduce the results before returning back to the caller.

This reduction will likely not be done on a single machine (unless the data to be reduced over is so small that sending it over a network would be inefficient.)

After the intial chunks have been reduced, we still need to reduce the results of the initial reduction, so we call our function again and either redistribute the initial results or simply locally-process chunks, depending on the size of the initial results array.

Parameters:
  • foo – function to reduce over data
  • data – a list of data to be reduced
  • port – a port by which to send over distributed operations
  • timeout – timeout, in seconds, that function should wait for chunks to be returned
Returns:

the reduced result (a single value!)
